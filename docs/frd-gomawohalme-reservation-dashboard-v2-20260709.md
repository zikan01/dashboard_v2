## 3. 대시보드 홈

### 기능
관리자와 직원이 가장 먼저 보는 화면이다. 이번 달 핵심 운영 상태와 오늘 확인해야 할 예약을 보여준다.

### 표시 항목

| 영역 | 내용 |
|---|---|
| KPI 카드 | 이번 달 예약 수, 방문 인원, 예상 매출, 다가오는 예약, 취소/변경 예약 |
| 오늘 방문 예약 | 오늘 방문 예정 예약 목록 |
| 다가오는 예약 | 가까운 방문일 기준 예약 목록 |
| 준비 알림 | D-day 기준 확인 항목 |
| 최근 반영 상태 | 마지막 반영 시각, 반영 건수, 성공/실패, **로컬 엑셀 저장 여부** |

### KPI 계산 기준

| KPI | 계산 방식 |
|---|---|
| 이번 달 예약 수 | 이번 달 예약 중 **취소를 제외한(확정+변경)** 예약 수 |
| 총 방문 인원 | 취소 제외 예약의 방문 인원 합계 |
| 예상 매출 | 취소 제외 예약의 결제금액 합계 |
| 다가오는 예약 | 오늘 이후 방문 예정인 확정/변경 예약 수 |
| 취소/변경 예약 | 이번 달 취소 또는 변경 상태 예약 수 |

> v1 라벨 "확정 예약"은 실제 로직(취소만 제외, 변경 포함)과 어긋나므로 **"이번 달 예약 수 / 취소 제외"**로 통일.

## 5. 예약 상세 화면

### 표시 항목

| 항목 | 설명 |
|---|---|
| 표시번호 | 자체 PK(GMW-…) |
| 예약번호 | 네이버 예약번호(있으면) |
| 예약자명 | 고객 이름 |
| 연락처 | 전체 연락처 표시 |
| 방문일 | 이용 시작일과 종료일 |
| 인원 | 방문 인원 |
| 옵션 | 전체 옵션 목록 |
| 결제금액 | 예약 결제금액 |
| 예약상태 | 확정/변경/취소 |
| 출처 | 엑셀/수집기/텍스트문의 |
| 유입경로 | 네이버 플레이스, 네이버 지도 등 |
| 정산 상태 | 수동 체크 |
| 세금계산서 상태 | 수동 체크 |
| 메모 | 대표/직원 메모 (**문자열 텍스트로 저장·표시**) |

> ⚠️ **버그 수정 명시**: v1 화면에서 메모가 `[object Object]`로 표시됨. 메모는 객체가 아니라 **문자열 값**만 저장·바인딩한다.

### 상태값
정산: 별도 확인 필요 / 확인 완료 / 해당 없음. 세금계산서: 별도 확인 필요 / 발행 완료 / 해당 없음.

### 기본값 규칙
확정/변경 예약은 두 상태를 `별도 확인 필요`로, 취소 예약은 `해당 없음`으로 생성한다.

### 소유권 규칙 (신규)
정산 상태·세금계산서 상태·메모는 **Supabase가 원본**이며, 엑셀 재업로드로 **덮어쓰지 않는다**. 이 필드들은 직원도 수정할 수 있다.

### 수정 이력
정산 상태, 세금계산서 상태, 메모가 변경되면 수정 이력을 기록한다.

## 9. 엑셀 업로드 화면 (🔑 관리자 전용)

### 업로드 흐름
1. **파일 선택 버튼**으로 로컬 엑셀 선택 (브라우저에서 SheetJS 파싱)
2. 파일 형식 확인
3. 필수 컬럼 검증
4. 예약 식별(네이버 예약번호 또는 자체 PK) 기준 파싱
5. **자체 PK 부여** + 병합 후보 탐색
6. 업로드 미리보기 생성
7. 사용자가 반영하기 클릭
8. **로컬 엑셀 저장 + Supabase 반영 동시 처리**
9. 결과 표시

### 검증 항목

| 항목 | 설명 |
|---|---|
| 파일 형식 | `.xlsx`, `.xls` |
| 예약자명 | 필수 |
| 연락처 | 필수 |
| 이용기간 | 필수 |
| 인원 | 필수 |
| 예약번호 | 있으면 저장(없으면 자체 PK만) |
| 옵션 | 있으면 저장 |
| 결제금액 | 있으면 매출 계산 |
| 예약상태 | 확정/변경/취소 판정 |

## 10. 업로드 미리보기 화면 (🔑)

### 표시 항목

| 항목 | 설명 |
|---|---|
| 전체 예약 수 | 파일에서 읽은 예약 수 |
| 신규 예약 수 | 새로 생성될 예약 수 |
| 업데이트 예약 수 | 기존 예약과 일치해 갱신될 예약 수 |
| **병합 후보 수** | 텍스트 문의건과 이름+연락처+방문일이 일치하는 후보 |
| 취소 예약 수 | 취소 상태로 반영될 예약 수 |
| 변경 예약 수 | 변경 상태로 반영될 예약 수 |
| 오류 수 | 저장할 수 없는 예약 수 |
| 오류 상세 | 컬럼 누락, 날짜 오류, 금액 오류 등 |

### 반영 원칙
- 같은 예약은 새로 만들지 않고 기존 예약을 업데이트한다.
- **병합 후보는 관리자가 확인 후 병합**한다(자동 병합 없음).
- **정산 상태·세금계산서 상태·메모는 업로드로 덮어쓰지 않는다.**

## 12. 텍스트 문의 화면 (신규, 🔑 관리자 전용)

### 기능
전화·문자·카톡으로 온 자유 텍스트 예약 문의를 입력·정리·승격한다.

### 흐름
1. 관리자가 문의 원문을 붙여넣기/입력 → `대기` 상태 저장(원문 보존)
2. 관리자가 방문일·인원·옵션·이름·연락처를 확인·수정
3. 병합 후보(기존 예약과 이름+연락처+방문일 일치)가 있으면 표시
4. "예약으로 승격" → 자체 PK 부여, 정식 예약 생성(또는 기존 예약에 병합)

### 상태

| 상태 | 의미 |
|---|---|
| 대기 | 아직 예약으로 확정 안 됨 |
| 확정 | 예약으로 승격됨 |
| 반려 | 예약 아님으로 처리 |

> v1은 수동 입력·정리. AI 자동 파싱은 v2(파싱 서비스 추후 결정), 도입 시에도 관리자 검수 후 승격.

## 13. 데이터 내보내기 화면 (신규, 🔑 관리자 전용)

### 기능
원하는 필드만 골라 엑셀로 내보내면서, 같은 데이터가 대시보드에도 반영되도록 한다.

### 구성

| 항목 | 설명 |
|---|---|
| 내보내기 프로필 | 포함할 컬럼 선택(표시번호·방문일·예약자·인원·옵션·금액·상태 등) 저장·재사용 |
| 필터 | 기간·상태 등으로 대상 예약 범위 지정 |
| 내보내기 실행 | 자체 PK가 포함된 엑셀을 로컬에 저장(`예약마스터_YYYYMMDD_HHmm.xlsx`) |
| 동시 반영 | 저장과 동시에 Supabase 대시보드 갱신 |

### 규칙
- 파일은 **날짜·시각별 새 파일**로 저장(덮어쓰기 금지, 이력 보존)
- 로컬 저장과 Supabase 반영은 한 묶음으로 처리, 실패 시 전체 롤백

## 18. 공통 오류 메시지

| 상황 | 메시지 예시 |
|---|---|
| 잘못된 파일 형식 | 네이버 예약 상세 엑셀 파일만 업로드할 수 있습니다. |
| 연락처 없음 | 연락처 컬럼을 읽을 수 없습니다. 파일 형식을 확인해 주세요. |
| 이용기간 오류 | 이용일 형식을 읽을 수 없습니다. |
| 저장 실패 | 예약 데이터를 저장하지 못했습니다. 다시 시도해 주세요. |
| 로컬 저장 실패 | 로컬 엑셀 저장에 실패했습니다. 다시 내보내기 하세요. |
| 반영 부분 실패 | 로컬 또는 온라인 반영 중 오류가 발생해 되돌렸습니다. 다시 시도해 주세요. |
| 수집 실패 | 네이버 로그인 상태를 확인한 뒤 다시 실행해 주세요. |
| 권한 없음 | 이 작업은 관리자만 할 수 있습니다. |
