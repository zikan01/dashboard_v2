# 솔라피 문자 발송 시스템 범위 축소 설계 (v3.0 → v3.1)

> 작성일: 2026-07-17
> 대상: `docs/20260717_solapi_system/` PRD·FRD·TRD 3종 + HTML 시안
> 상태: 사용자 승인 완료 (2026-07-17)

## 배경

기존 v3.0 문서는 "예약 고객 자동 안내 + 고객 사진 전달 + 만족도 조사 + 네이버 후기 연결"을 포함하는
예약 운영 대시보드 확장 문서였다. 협업 범위 조정에 따라 본인 담당분을
**솔라피(SOLAPI) API 기반 문자 발송 시스템**으로 좁힌다.

## 확정 결정 (사용자 답변 기준)

1. **채널: 문자(SMS/LMS)만.** 알림톡(카카오)·이메일·동시 발송·알림톡 실패 시 문자 대체 로직 전부 제거.
2. **기존 파일 제자리 수정.** 버전 v3.1, 작성일 2026-07-17, 파일명 `...v3.1-20260717.md`로 git mv.
3. **HTML 시안**: 동료 담당 화면(대시보드 홈·예약 목록·예약 상세)은 그대로 유지.
   후기·만족도·사진·고객용 페이지 화면 삭제. 본인 담당 6개 페이지 내 알림톡·이메일 UI를 문자 기준으로 정리.
4. **협업 경계**: 동료 = 예약 CRUD·대시보드 홈·예약 목록·예약 상세. 본인 = 발송 시스템 6개 페이지 + 발송 엔진.

## 범위 정의

### 유지 (본인 담당)

- 페이지 6개: 자동안내설정, 메시지 템플릿, 솔라피 설정, 발송 일정, 발송 이력, 실패 관리
- D-7·D-3·D-1·당일 자동 안내 (문자)
- 문자 템플릿·변수 치환·미리보기 (SMS/LMS 자동 판별)
- 발송 일정 생성·재계산 (예약 생성·변경·취소 감지)
- 발송 이력·비용 집계
- 실패 관리·수동 재발송 (발송 일정/실패 관리 페이지에서 수행)
- 중복 방지(dedupe key), Cron, Job Claim, 발송 직전 재검증
- SOLAPI Webhook (문자 상태 수신)
- 솔라피 설정: API Key/Secret, 발신번호, 잔액, 테스트 발송
- 감사 로그, 모니터링, Dry Run→Allowlist→파일럿→전체 출시 단계

### 제거

- 알림톡(카카오 채널, 템플릿 승인, 대체·동시 발송), 이메일
- 고객 사진 전달·30일 자동 삭제·사진 Storage·고객 사진 페이지
- 만족도 조사·설문 테이블·만족도 결과 화면
- 네이버 후기 연결
- 고객 공개 페이지·고객 접근 토큰·고객 이벤트
- 서비스 완료 및 발송 모달·서비스 상태 테이블

### 동료 화면과의 연계 (인터페이스 계약)

- 발송 시스템은 예약 테이블을 **읽기 전용**으로 참조한다.
- 예약 상태·방문일·연락처 변경을 감지해 발송 일정을 재계산한다.
- 기존 FRD의 "예약 상세 자동 안내 탭·예약별 덮어쓰기·수동 발송"은 동료 화면 수정이 필요하므로 범위에서 제외.
  수동 발송·재발송 기능은 발송 일정/실패 관리 페이지로 이동.

## 문서별 수정 방침

- **PRD**: 목표에서 사진·만족도 삭제, 발송 채널을 문자 단일로 축소, 12~15장(사진·만족도·후기) 삭제,
  관리자 화면을 6개 페이지로 교체, 성공 기준·최종 확정 결정 정리, 담당 범위 섹션 신설.
- **FRD**: 화면 구성을 6개 페이지로 축소, 홈·예약 목록·상세 탭 정의는 삭제하고 "동료 담당 화면과의 연계"
  참고 섹션으로 대체, 템플릿에서 알림톡·이메일 제거, 사진·설문 화면 삭제, 상태값·오류 문구를 문자 기준으로 정리.
- **TRD**: 신규 테이블 중 서비스 상태·사진·토큰·설문·고객 이벤트 삭제, Delivery를 SMS 단일 채널로 단순화
  (dispatch_mode·alimtalk/email 필드 제거), SOLAPI 처리를 문자만으로 축소, 이메일·사진·설문·공개 API 장 삭제,
  환경변수·Migration·구현 순서 재정리.
- **HTML**: 위 HTML 방침대로.
