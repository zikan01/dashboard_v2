## 2. 이슈 상세

### #1 🟠 High — KPI "준비물 미등록 옵션 0건"의 측정 수단이 선택 기능(Tier 2)

- **메타**: PRD 본문 §7.2 ↔ PRD 핸드오프 §2.1(A-009 선택) ↔ FRD §2.2(Tier 2 "여유 시")
- **문제**: KPI 측정 방법이 "옵션 업로드 화면 미등록 목록"인데, 그 목록(A-009)은 v1 확정 범위가 아님. A-009가 빠지면 이 KPI는 측정 불가.
- **왜 중요한가**: 측정 불가능한 KPI는 배포 후 "됐는지 안 됐는지 모르는 목표"가 된다.
- **수정 방향**: 측정 수단을 v1 필수 기능으로 교체. 내보내기(A-004)가 미등록 옵션을 `옵션명: (미등록)`으로 출력하므로 이걸 측정 수단으로 사용.
- **수정안**: PRD §7.2 측정 방법을 "내보내기 파일의 '(미등록)' 표기 건수"로 변경. → **반영 완료**
- **재지시 프롬프트**: "PRD §7.2의 '준비물 미등록 옵션' KPI 측정 방법을 A-009 화면 의존에서 A-004 내보내기 '(미등록)' 표기 집계로 바꿔줘."

### #2 🟡 Medium — 사이드바 메뉴 순서가 문서 3곳에서 불일치

- **메타**: 요구사항 원문(`옵션 및 준비물 수기 입력.md`: 텍스트 문의 다음) ↔ PRD 핸드오프 §6·FRD 핸드오프 §1(엑셀 업로드 다음) ↔ 현재 코드(엑셀 업로드/텍스트 문의/내보내기/이력)
- **문제**: 신규 메뉴 "옵션 업로드"의 삽입 위치가 문서마다 다름.
- **왜 중요한가**: 사이드바는 협업자와의 병합 접점이라, 구현 후 위치를 바꾸면 충돌을 두 번 겪는다.
- **수정 방향(권고)**: **엑셀 업로드 바로 다음**으로 통일 — 업로드 계열 기능이 붙어 있는 것이 사용 흐름(업로드→옵션 등록)과 맞음.
- **수정안**: 요구사항 원문 md의 순서 표기를 PRD/FRD와 일치하도록 수정. → **반영 완료** (이견 있으면 한 곳만 바꾸면 됨 — 세 문서 모두 순서를 명시한 곳은 2곳)
- **재지시 프롬프트**: "옵션 업로드 메뉴 위치를 '엑셀 업로드 다음'으로 요구사항 md·PRD·FRD 모두 통일해줘."

### #3 🟡 Medium — PRD 핸드오프 §9 파일 맵에 `/api/preparations` 누락

- **메타**: PRD 핸드오프 §9 ↔ FRD 핸드오프 §7·§10 ↔ TRD 핸드오프 §7
- **문제**: FRD/TRD에는 신규 API 3개(preparations·settings·import 확장)가 있는데 PRD 핸드오프 신규 파일 목록에는 settings만 있음.
- **왜 중요한가**: PRD 핸드오프를 단독으로 Claude Code에 입력하면 준비물 API 없이 화면만 만들려고 시도한다.
- **수정안**: PRD 핸드오프 §9 신규 파일에 `app/api/preparations/route.ts` 추가. → **반영 완료**
- **재지시 프롬프트**: "PRD 핸드오프 §9 신규 파일 목록에 api/preparations/route.ts를 추가해줘."

### #4 🟢 Low — `app_settings.value_encrypted`에 토큰 "해시" 저장 (칼럼명 의미 불일치)

- **메타**: TRD 핸드오프 §2
- **문제**: 비밀번호는 암호문(복호화 가능), 토큰은 해시(복호화 불가)인데 같은 칼럼 `value_encrypted`에 저장 — 이름이 실체와 어긋남.
- **수정안**: 칼럼명을 `value`로 두고 주석으로 형식 구분(현행 SQL 주석에 이미 구분 있음). 구현 시 칼럼명만 `value`로 변경 권고. → TRD 핸드오프 §2에 주석 보강 **반영 완료**
- **재지시 프롬프트**: "app_settings 칼럼명을 value로 바꾸고 형식(암호문/해시)을 주석으로 구분해줘."

### #5 🟢 Low — 수집기 성공률 KPI(90%)의 측정 주체 미지정

- **메타**: PRD §7.2 ↔ TRD 핸드오프 §6 (로그는 대표 PC 로컬 파일)
- **문제**: 측정 방법이 "수집기 로그"인데 로그는 대표 PC에만 있어 개발자가 원격 집계 불가.
- **수정안**: v1은 "대표에게 실패 시 보고 요청 + 월 1회 로그 파일 회수"로 운영 측정(문서 변경 없이 운영 절차로 처리). 자동 집계는 v2 항목. → 액션 아이템에만 기재, 문서 무변경.
