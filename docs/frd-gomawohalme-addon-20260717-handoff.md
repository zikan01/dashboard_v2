### 매칭 규칙 (확정 — PRD 핸드오프 §3.3 상속)

```
normalize(s) = lower(remove_spaces(s))
옵션 1건에 대해:
  후보 = active preparation_items 중 normalize(option_name)이 normalize(option_keyword)를 포함하는 것
  포함 관계 중복(예: "계곡" ⊂ "계곡체험") → 가장 긴 keyword 1개만 채택
예약 1건 = 옵션별 결과를 옵션 단위로 그룹 표시 (합치지 않음)
내보내기 형식 = "옵션명: 항목, 항목 / 옵션명: (미등록)"
```

## 5. 인수 기준 (AC) — 화면별

### S-A01 옵션 업로드
- [ ] 옵션 등록 후 1초 이내 목록에 반영
- [ ] 등록 직후 해당 옵션을 가진 기존 예약 상세에 준비물 표시 (새로고침 기준)
- [ ] 동일 옵션명("바베큐" vs " 바베큐 ") 저장 시 병합 모달 표시
- [ ] 병합 확인 시 준비물 = 합집합, 중복 없음
- [ ] 삭제 후 예약 상세에서 해당 옵션이 "준비물 미등록"으로 전환
- [ ] 직원 계정 URL 직접 접근 시 대시보드로 리다이렉트
- [ ] 375px 뷰포트에서 가로 스크롤 없음

### S-A02 업로드 확장
- [ ] 비밀번호 등록 상태에서 암호화 엑셀 업로드 → 에러 없이 기존 미리보기 도달
- [ ] 미등록 상태에서 암호화 엑셀 → 지정 에러 메시지 + 설정 카드 스크롤
- [ ] 잘못된 비밀번호 → "파일 비밀번호가 맞지 않습니다", 데이터 무변화
- [ ] 저장된 비밀번호는 응답·화면 어디에도 평문 노출 없음
- [ ] 일반(비암호화) 엑셀 업로드는 기존과 완전 동일 동작 (회귀 없음)

### S-A03 상세 확장
- [ ] 옵션 "바베큐 4인 세트" 예약에 키워드 "바베큐" 준비물 표시 (부분 일치)
- [ ] "계곡"·"계곡 체험" 동시 등록 시 옵션 "계곡 체험"에는 긴 키워드 것만 표시
- [ ] 미등록 옵션 → 그레이 배지, 관리자에게만 등록하기 링크
- [ ] 준비물 렌더로 기존 상세 레이아웃 요소가 이동·변형되지 않음 (기존 작업물 보호)

### S-A04 내보내기 확장
- [ ] 준비물 필드 선택 시 출력 형식 "옵션명: 항목, 항목 / …" 준수
- [ ] 미선택 시 기존 내보내기와 바이트 단위 동일 동작 (회귀 없음)

### C-A01 수집기
- [ ] Happy Path: 더블클릭 → 3분 이내 "N건 반영" (예약 100건 기준)
- [ ] 로그인 풀림 → 지정 안내 후 중단, 데이터 무변화
- [ ] 버튼 못 찾음 → 선택자 안내 후 중단
- [ ] 다운로드 60초 초과 → 시간 초과 안내 후 중단
- [ ] 복호화 실패 → 원본 보존 + 안내
- [ ] 성공 시 업로드 이력에 방식 "로컬 수집기" 배치 1건 기록
- [ ] 수집기 실행 파일·설정에 네이버 자격증명 없음 (코드 리뷰)

## 8. 5개 상태 상세 — S-A01 (신규 화면만 정밀 기술)

| 상태 | 보이는 것 | 가능한 액션 | 시스템 처리 |

| Success | 폼 + 목록 표 | 등록·수정·삭제 | 저장 후 목록 재조회 |
| Empty | 폼 + "등록된 옵션이 없습니다" + 예시 플레이스홀더 | 등록 | 첫 저장 시 Success 전환 |

| Disabled | (직원) "관리자 전용 화면입니다" | - | 대시보드 리다이렉트 |
