# Claude Code 지시서 — 수집기 연결 토큰 UI 제거

## 배경
자동 다운로드 수집기(collector)를 사용하지 않기로 확정했다. 대표는 네이버에서 암호화 엑셀을 직접 받아 대시보드에 수동 업로드하고, 서버가 등록된 비밀번호로 자동 복호화한다.
**"네이버 파일 비밀번호" 등록/복호화 기능은 그대로 유지**하고, **"수집기 연결 토큰" 관련 UI만 제거**한다.

## 변경 요청

1. `src/app/(dashboard)/upload/page.tsx`에서 "수집기 연결 토큰" 섹션 전체 제거
   - 소제목 "수집기 연결 토큰", 설명 문구("대표 PC의 수집기(config.json)에 넣을 전용 토큰입니다…"), "수집기 토큰 발급" 버튼과 발급 결과 표시 UI, 관련 상태(useState)·핸들러·fetch 호출 모두 삭제
   - "네이버 파일 비밀번호" 카드와 저장 기능은 **유지**
   - 파일 선택/업로드/미리보기/반영/되돌리기 등 기존 기능은 **손대지 않음**

2. `src/app/api/settings/route.ts`에서 토큰 발급(issueToken) 분기 제거
   - 토큰 발급 로직, `collector_token_hash` 저장 코드 삭제
   - 비밀번호 저장(password)·상태 조회(passwordSet) 로직은 **유지**
   - GET 응답에서 `tokenIssuedAt` 필드 제거 (남아도 무방하지만 미사용이면 정리)

3. 사용하지 않게 되는 관련 코드 정리 (있는 경우에만)
   - `hashToken` 등 토큰 전용 유틸이 다른 곳에서 안 쓰이면 삭제
   - `app_settings`의 `collector_token_hash` 키는 마이그레이션에서 굳이 지우지 않아도 됨(스키마 변경 최소화)

## 유지해야 할 것 (건드리지 말 것)
- 네이버 파일 비밀번호 등록/저장 (`forest_unnie`)
- 업로드 시 암호화 엑셀 자동 복호화 (`/api/import`의 복호화 로직, `lib/crypto.ts`의 decrypt)
- 옵션 업로드·준비물 매칭 기능 전체
- 기존 업로드/미리보기/반영 흐름

## 완료 조건
- 엑셀 업로드 화면에 "네이버 파일 비밀번호" 카드만 남고 "수집기 연결 토큰" 섹션은 사라짐
- 암호화 샘플 엑셀 수동 업로드 → 비밀번호 입력 없이 복호화되어 미리보기까지 정상 (회귀 없음)
- 로컬 `npm run build` 통과 후 커밋·푸시

## 참고
- collector/ 폴더는 이번에 삭제하지 않아도 됨(웹 빌드에서 이미 제외됨). 완전히 안 쓸 거면 별도로 폴더 삭제 요청 가능.
