## 0. 문서 목적

본 문서는 솔라피(SOLAPI) 기반 문자 발송 시스템을 화면과 사용자 동작 단위로 정의한다.

담당 범위는 아래 6개 화면과 발송 엔진이다. 대시보드 홈, 예약 목록, 예약 상세는 동료 담당이며 본 문서에서 정의하지 않는다.

## 3. 동료 담당 화면과의 연계

발송 시스템은 동료가 담당하는 예약 데이터를 읽기 전용으로 참조한다.

- 예약 생성·변경·취소 시 발송 일정을 자동 생성·재생성·취소한다.
- 고객명, 전화번호, 방문일, 예약 상태는 예약 테이블 값을 사용한다.
  예약 상태는 `confirmed`(확정)·`changed`(변경)일 때만 발송 대상이고 `cancelled`(취소)는 발송을 중단한다.
- 전화번호 수정은 동료 담당 화면에서 수행하며, 발송 시스템은 발송 직전 서버에서 형식을 재검증한다.
- 예약 상세 내 자동 안내 탭, 예약별 덮어쓰기, 예약 상세에서의 수동 발송은 본 범위에서 제외한다. 수동 발송·재발송은 발송 일정과 실패 관리 화면에서 수행한다.

전화번호 처리 기준:

- 공백·하이픈 제거 후 국내 휴대전화 형식 검증
- 유효하지 않으면 해당 예약은 발송 대상에서 제외하고 실패 관리에 `전화번호 오류`로 표시
- 네이버 엑셀 유입 데이터에는 별표로 마스킹된 번호(`****...`)가 존재한다.
  마스킹 번호는 유효하지 않은 번호로 처리하며, 발송 일정 화면에서 `연락처 확인 필요`로 표시한다

## 4. 자동 안내 설정

### 4.1 전체 설정

- 자동 안내 전체 활성화
- 기준 시간대
- 신규 예약 일정 자동 생성
- 변경 시 자동 재생성
- 취소 시 미발송 작업 자동 취소
- 테스트 예약 제외
- 전화번호 누락 경고

### 4.2 단계별 카드

D-7·D-3·D-1·당일 각 카드:

- 활성화
- 발송 시간
- 문자 템플릿
- 예상 문자 유형(SMS·LMS)과 비용
- 미리보기
- 테스트 발송

문자 템플릿이 지정되지 않은 단계는 활성화할 수 없다.

## 12. 중복 방지

동일 예약에서 다음 조합은 한 번만 생성한다.

- 예약 ID
- 단계
- 기준 방문일
- 규칙 버전

중복 검사는 예정·처리 중·성공 상태에만 적용한다. 취소되거나 제외된 일정은
중복으로 계산하지 않으므로 재생성할 수 있다. 수동 재발송은 원본 작업과
연결된 별도 이력으로 저장하며 중복 검사와 충돌하지 않는다.

## 13. 주요 오류 문구

- `고객 전화번호가 없어 문자를 발송할 수 없습니다.`
- `전화번호 형식이 유효하지 않습니다.`
- `이 단계의 안내가 이미 발송되었습니다. 다시 발송하시겠습니까?`
- `SOLAPI 잔액이 부족합니다.`
- `발신번호가 등록되지 않았습니다.`
- `문자 템플릿이 지정되지 않아 활성화할 수 없습니다.`

## 16. 수용 기준

- D-7·D-3·D-1·당일 문자 자동 발송이 동작한다.
- 예약 생성·변경·취소에 따라 일정이 정확히 관리된다.
- 동일 예약·동일 단계 중복 발송이 차단된다.
- 발송 결과와 비용을 확인할 수 있다.
- 실패 작업을 재발송·제외 처리할 수 있다.
- SMS·LMS가 본문 길이에 따라 자동 판별된다.
- 직원은 조회만 가능하고 설정·발송은 대표만 가능하다.
