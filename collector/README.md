# 고마워할매 로컬 수집기

네이버 예약자관리(partner.booking.naver.com)에서 **상세 엑셀을 자동으로 내려받아**
대시보드(`/api/import`, auto_apply)에 반영하는 대표 PC용 프로그램입니다.
웹앱과 완전히 분리된 폴더이며 별도의 `package.json`을 가집니다.

## 동작 (6단계)

1. `config.json` 로드
2. CDP(127.0.0.1:9222) 접속 — 실패 시 크롬을 수집 전용 프로필로 실행
3. 예약자관리 페이지 이동 → 로그인 풀림 감지 시 안내 후 중단 (자격증명 입력 없음)
4. **"상세 내려받기"** 버튼 클릭 (기간은 기본값 '이용일 · 한달' — 조작하지 않음)
5. 다운로드 폴더 감시 (`*예약자관리*.xlsx`, 60초)
6. 업로드 — 암호화 파일은 서버가 등록된 비밀번호로 복호화
   (config의 `excelPassword`를 채우면 로컬에서 복호화 후 업로드)

종료 코드: `0` 성공 · `1` 일반 실패 · `2` 로그인 필요 · `3` 버튼 못 찾음 ·
`4` 다운로드 시간 초과 · `5` 복호화 실패 · `6` 업로드 실패

## 개발

```bash
cd collector
npm install
npm run build     # tsc → dist/
npm start         # node dist/index.js
```

## 배포 (대표 PC)

```bash
npm run package   # @yao-pkg/pkg → collector.exe (node20-win-x64)
```

대표 PC에 아래를 복사:

```
collector/
├── collector.exe
├── config.json          ← 발급받은 수집기 토큰 기입 (대시보드 → 엑셀 업로드 → 설정 카드)
└── scripts/
    ├── 1_브라우저_열기.bat   ← 바탕화면 바로가기 1 (최초 1회 네이버 로그인)
    └── 2_예약_가져오기.bat   ← 바탕화면 바로가기 2 (평소 이것만 더블클릭)
```

## 보안

- 네이버 아이디·비밀번호를 저장하거나 입력하지 않습니다. 로그인은 대표가 브라우저에서 직접.
- 디버그 포트는 127.0.0.1 전용, 수집 전용 프로필로 상시 브라우저와 격리됩니다.
- 로그(`logs/YYYY-MM-DD.log`)에는 단계·건수만 기록하며(개인정보 금지) 30일 지나면 자동 삭제됩니다.
- `config.json`에 실제 토큰을 넣은 채 git에 커밋하지 마세요.

## 네이버 화면이 바뀌었을 때

버튼을 못 찾으면 `config.json`의 `selectors.detailDownloadBtn`만 수정하면 됩니다
(코드 수정·재빌드 불필요). 기본값은 텍스트 기반 선택자
`button:has-text("상세 내려받기")` — 옆의 "내려받기" 버튼과 혼동하지 않도록
반드시 "상세 내려받기" 전체 텍스트를 사용하세요.
